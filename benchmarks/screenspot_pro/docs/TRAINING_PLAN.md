# 本地 Qwen3-VL(4B/8B)训练方案:让"我们的方法"真正产生增益

## 0. 目标澄清(务必先对齐)

目标不是单纯把 4B/8B 模型的 SSPro 分数堆到 80%——那件事主要靠训练数据本身就能做到,跟我们的
方法(证据脚手架 + 迭代裁剪)无关。**真正要证明的是:经过恰当训练之后,我们的方法能在这个小模型
身上产生纯 SFT 拿不到的增量**。所以整个方案的核心不是"SFT 到 80%",而是"SFT 冷启动 + agentic RL
教会它用我们的工具",最后用三组对照把"方法的增量"独立测出来。

如果只是想要一个能打 80% 的单发模型,直接冲纯 SFT 就够,不需要下面 RL 那部分——但那样论文里就
证明不了"我们的方法有价值",只能证明"我们的训练数据有价值"。两者是不同的命题,选之前先想清楚
要证明哪个。

硬件:远程 Pro 6000(Blackwell,96GB)。QLoRA 微调 4B/8B 都绰绰有余;agentic RL 阶段的 rollout
是主要瓶颈,不是显存。

---

## 阶段 0:诊断基线模型(训练之前,别跳过)

在写任何一行训练代码之前,先用现成的 SOP 给**没训练过的** Qwen3-VL-4B/8B(base/instruct 权重)
测一遍:

1. 格式消融(baseline50,~50题×4格式,单发无提示)——**不要假设**它跟 qwen3.7-plus 用同一套
   `point_2d`+[0,1000] 约定,虽然大概率是(同一家族、同一套官方 grounding 训练协议),但必须实测
   确认,复用 `probe_m3_format.py` 的写法换成本地推理调用即可。
2. 三探针诊断(thinking敏感性/证据敏感性/格式方差)——搞清楚这个基座模型本身的"性格"更偏专训头
   还是通用推理。如果它天生就偏通用推理(不太可能,但不能排除),那 RL 阶段会容易很多;如果天生
   偏专训头,说明需要 SFT 阶段更用力地把它"掰"向使用证据的方向。

这一步全程本地推理,不花 API 钱,几个小时之内能出结果,能避免训练配方建立在错误假设上。

---

## 阶段 1:SFT 冷启动——教会格式,教会"候选证据是存在的"

**目的不是让准确率立刻拉满**,是让模型进入一个"能可靠输出规范坐标 + 见过证据增强提示"的起点状态,
给阶段 2 的 RL 一个能站稳的地基。

### 数据构造(最大的不确定性在这里,不在算法)

- 基础样本:截图 + 指令 + GT 中心点,按阶段0 确认的原生格式(大概率 `point_2d`+[0,1000])编码。
- **关键设计**:30~50% 的样本在 prompt 里插入候选框列表(复用现成的 `detect_components` +
  `active_localization.build_candidates`),而且**候选里要故意混入错的/不完整的**,不能全给对的
  候选——目的是从训练分布起点就教会模型"候选是参考,不是标准答案",而不是像 qwen3.7-plus 那样
  从没在训练里见过这种模式,测试时硬塞给它当然会懵。其余50~70%样本不给候选,保证它离开候选也能
  独立定位,不产生依赖。
- 数据量:没有已验证的数字,这是最大风险点——建议先用小规模(几百到一两千条)跑通训练管线本身,
  再视 loss/验证集表现决定要不要扩量,不要一开始就赌大规模标注。
- 覆盖面:尽量覆盖 SSPro 的~26个应用类别,避免只学会某几个软件的截图分布。

### 训练方法

- LoRA/QLoRA(不建议一上来就全量微调,数据量不确定时 LoRA 更容易快速迭代、更容易回滚)。
- 框架:LLaMA-Factory 或 Unsloth,两者都确认支持 Qwen-VL 系列的多模态 LoRA/QLoRA(见前面调研)。
- 冻结/解冻策略:视觉编码器通常建议先冻结,只训 LLM 部分 + 投影层,除非确认视觉编码器本身也是
  瓶颈——先用最省资源的配置跑通,再按需放开。

### 验收标准

不要求这一阶段就到 80%。验收看两点:①单发准确率有没有明显超过基座模型的诊断基线(阶段0数据)
②快速抽样检查——给它候选证据时,输出有没有变得"合理地被影响"(而不是完全忽略,也不是照抄候选)。

## 阶段 1.5:训练后复测诊断(承上启下,别跳)

用阶段0同一套三探针,给 Stage-1 SFT 后的 checkpoint 重新测一遍:

- 如果证据敏感性(探针B)变正了(有证据比没证据准)——说明 SFT 数据设计生效了,这个模型现在
  真的在学着用证据,阶段2 的 RL 大概率有戏。
- 如果证据敏感性还是负的(SFT 把它训成了新的专训头)——说明候选证据在训练数据里占比/构造方式
  需要调整,重新回阶段1 调整数据配比,不要直接跳去做 RL(冷启动状态不对,RL 大概率训不动)。

这一步是个分岔口,决定要不要继续往下走。

---

## 阶段 2:Agentic RL——在我们的 harness 环境里训练"要不要用工具"的策略

**这一步才是真正回答"我们的方法有没有用"的实验**,前面 SFT 只是让模型具备基本能力,这里才是让
它学会"什么时候该裁剪重看、什么时候该信证据、什么时候该坚持自己的初判"这个决策本身。

- **环境**:把现有迭代缩放 harness(或简化版)包装成 RL 环境——动作空间大致是"当前视图直接commit"
  /"请求裁剪到某个区域"/"请求查看候选证据",奖励=最终点是否落在 GT 框内(稀疏二值奖励,可以加一点
  按调用次数的小惩罚,防止无限拖着不收敛)。
- **算法**:多轮工具调用+稀疏奖励,适合 GRPO(DeepSeek-R1 那套训练法用的)或 PPO,每个样本需要
  多次 rollout 才能拿到有效梯度信号——这一步的计算成本远高于 SFT,是整个方案里最贵、最不确定的
  部分。
- **起点**:必须从阶段1 的 SFT checkpoint 开始训,不要从没训练过的基座直接上 RL——冷启动做 agentic
  RL 大概率不稳定、采样效率极低。
- **建议**:先在一个很小的子集(几十到上百题)上把 rollout 管线跑通、确认奖励设计真的能产生学习
  信号,再考虑扩大规模——这是工程上最容易卡住的一步,不要一开始就上全量数据。

---

## 阶段 3:评测——三组对照,分离出"方法的增量"

在一个没参与训练的 SSPro 留出集上,同时测三种配置:

| 配置 | 说明 | 预期 |
|---|---|---|
| (i) Stage-1 SFT,单发 | 纯 SFT 出来的定位能力 | 基线 |
| (ii) Stage-1 SFT + 我们的 harness(推理时直接套,没专门训过怎么用) | 测"不训练直接套用方法"够不够 | 按我们的理论,大概率持平或略负(重演 qwen 的故事) |
| (iii) Stage-2 RL + 我们的 harness | 真正训练过怎么用工具 | 应该明显超过 (i),且理想情况下比 (ii) 高出更多 |

**(iii) 比 (i) 涨多少,就是"我们的方法"的真实贡献;(iii) 比 (ii) 高出的部分,才是"训练怎么用
工具"这件事本身的价值(排除了"是不是随便套一下都会涨"这个混淆)。** 这个三组对照是整个方案的
核心产出,论文里"方法有没有用"这句话就靠这张表撑住。

---

## 风险清单(诚实列出,别到时候被审稿人问住)

- **数据量/质量是最大不确定性**,现在完全没有验证过多少条训练数据能撑起一个可用的 SFT 起点。
- **agentic RL 的训练稳定性和采样效率**是真实的研究难题,不是工程细节,不能假设"训一训就收敛"。
- **80% 这个数字本身不是核心指标**,核心指标是 (iii) 相对 (i)/(ii) 的增量;如果最终绝对分数到不了
  80% 但增量显著,依然是一个有效的结果,不要把叙事绑死在那个数字上。
